Design

Architecture — Three-Layer Design

PyGeoVision never reimplements PyGeoFetch or GeoAI. All data operations delegate to PyGeoFetch. All AI operations delegate to GeoAI. PyGeoVision is the integration layer that makes them work seamlessly together.

LayerPackageRoleAccess via
🛰️ DatapygeofetchSearch & download from 22+ providers — auth, caching, parallel downloads, post-processing, YAML pipeline orchestration, STAC 1.0 outputclient.search() · client.download() · client.data.*
🤖 AIgeoai-pyFull AI stack — segmentation, detection, classification, change detection, SAM, foundation models (Prithvi, DINOv3), embeddings, cloud masking, super-resolution, ONNX export, VLM captioningclient.geoai.* (lazy-loaded)

What PyGeoVision adds on top

🔗
Unified API
One PyGeoVision() client exposes everything — no need to import or configure PyGeoFetch and GeoAI separately. Lazy-loading means GeoAI only imports when you first access client.geoai.
⚡

One call from bounding box to finished GeoJSON/GeoTIFF: client.pipeline("building_footprints", bbox=..., date="2024-06"). PyGeoFetch fetches the data, GeoAI runs the model, PyGeoVision glues them together.
🏋️
Own Training Stack
14 registered model architectures, GeoTrainer (6 geospatial losses), TiledInference with Gaussian blending, 7 auto-labelers, ONNX export — all built on top of, but separate from, GeoAI's inference stack.
🏷️
7 Auto-Labelers
Generate training labels automatically from OSM, Microsoft Buildings, Google Open Buildings, ESA WorldCover, Google Dynamic World, SAM, and foundation models — zero manual annotation for common classes.
🖥️
Full CLI
The pygeovision CLI covers everything: data search/download, AI inference, model training, pipeline orchestration, credential management, cache, and status — all in one tool.
📋
YAML Pipelines

Getting Started

Up and running in 5 minutes

10 providers work immediately with no credentials. Free data includes Sentinel-2, Landsat, NAIP, and more via Microsoft Planetary Computer, AWS Earth, and Element84.

1
Install
# Core + full GeoAI stack pip install "pygeovision[geoai]" # Add raster / vector processing pip install "pygeovision[geo]" # Everything pip install "pygeovision[all]" # Verify pygeovision status
2
Add credentials (optional — free providers work immediately)
# CLI pygeovision data auth add usgs \ --username USER --password PASS pygeovision data auth add planet \ --api-key PL-xxxxx pygeovision data auth add copernicus \ --client-id ID --client-secret SECRET # Python client.add_credentials("usgs", username="u", password="p")
3
Search satellite data
pygeovision data search \ --bbox "-0.15,51.47,-0.10,51.52" \ --date 2024-06 \ --providers planetary_computer \ --cloud-max 10 \ --output results.geojson
4
Download + run GeoAI
pygeovision data download \ --from-search results.geojson \ --output ./data/ --parallel 4 # Or: one-command E2E pipeline pygeovision pipeline building_footprints \ --bbox "-0.15,51.47,-0.10,51.52" \ --date 2024-06 \ --output ./results/

Five-minute Python walkthrough

five_minute_start.py
import pygeovision as pgv

# ── 1. Initialise ─────────────────────────────────────────────────────────
client = pgv.PyGeoVision()
print(client)
# → PyGeoVision(v1.0.0, pygeofetch=✓, geoai=✓)

# ── 2. Add credentials (open providers need no credentials) ───────────────
client.add_credentials("usgs",       username="user",    password="pass")
client.add_credentials("planet",     api_key="PL-xxxx")
client.add_credentials("copernicus", client_id="id",     client_secret="s")

# ── 3. Search 22+ providers via PyGeoFetch ────────────────────────────────
results = client.search(
    bbox=(-0.15, 51.47, -0.10, 51.52),          # London, WGS84
    date_range=("2024-06-01", "2024-06-30"),
    providers=["planetary_computer", "copernicus"],
    cloud_cover_max=10,
    max_results=20,
    sort_by="cloud_cover",
)
for r in results[:3]:
    print(r)
# [planetary_computer] Sentinel-2C | 2024-06-03 | cloud=0% | score=0.99

# ── 4. Download + post-process via PyGeoFetch ─────────────────────────────
downloads = client.download(
    results[:3], output_dir="./sentinel2/",
    parallel=4, verify_checksum=True,
    post_process=["reproject:EPSG:4326", "cog"],
)

# ── 5. GeoAI: segment buildings ──────────────────────────────────────────
client.geoai.segment.buildings(
    downloads[0].path,
    output_vector="buildings.geojson",
    confidence_threshold=0.5,
)

# ── 6. GeoAI: change detection ────────────────────────────────────────────
client.geoai.change.detect(
    "scene_2020.tif", "scene_2024.tif",
    output_path="changes.tif",
)

# ── 7. End-to-end pipeline (search + download + AI in one call) ───────────
result = client.pipeline(
    "building_footprints",
    bbox=(-0.15, 51.47, -0.10, 51.52),
    date="2024-06", output_dir="./results/",
)
print(result.output_path)  # ./results/building_footprints/prediction.tif
print(result.stats)        # {"buildings_detected": 1847, "coverage_pct": 0.312}

All credentials are stored in your system keyring (macOS Keychain, Windows Credential Manager, Linux Secret Service) via PyGeoFetch — never in plain-text files. Environment variables are fully supported for CI/CD.

Satellite Data — PyGeoFetch

Searching Satellite Data

Federated search across all configured providers simultaneously. Results are deduplicated, scored by cloud cover and recency, and returned as STAC 1.0-compliant objects.

search_examples.py
import pygeovision as pgv
client = pgv.PyGeoVision()

# ── Basic search — free providers, no credentials ────────────────────────
results = client.search(
    bbox=(-74.1, 40.6, -73.7, 40.9),
    date_range=("2024-01-01", "2024-06-01"),
    providers=["planetary_computer", "aws_earth", "element84"],
    cloud_cover_max=10,
    max_results=50,
    sort_by="cloud_cover",
)
for r in results[:3]:
    print(r)  # [planetary_computer] Sentinel-2B | 2024-02-18 | cloud=2%

# ── By satellite shortcut ─────────────────────────────────────────────────
results = client.search(
    bbox=..., date_range=..., satellite="sentinel-2",
)

# ── By STAC collection ────────────────────────────────────────────────────
results = client.search(
    bbox=..., date_range=...,
    collections=["sentinel-2-l2a", "landsat-c2-l2"],
)

# ── CQL2 advanced filter (Planetary Computer, Element84) ─────────────────
results = client.search(
    bbox=..., date_range=...,
    providers=["planetary_computer"],
    cql2_filter="eo:cloud_cover < 5 AND platform = 'sentinel-2b'",
)

# ── SearchResult properties ───────────────────────────────────────────────
r = results[0]
r.id           # 'S2C_MSIL2A_20240603T153811_R001'
r.provider     # 'planetary_computer'
r.satellite    # 'Sentinel-2C'
r.date         # '2024-06-03'
r.cloud_cover  # 0.0
r.bbox         # (-0.15, 51.47, -0.10, 51.52)
r.score        # 0.99
r.resolution_m # 10.0
r.is_sar       # False
r.to_stac_item()   # pystac Item object

Downloading Satellite Data

Parallel, resumable downloads with SHA256 verification, exponential backoff retry, bandwidth throttling, band selection, and a composable post-processing chain.

download_examples.py
# ── Basic download ────────────────────────────────────────────────────────
downloads = client.download(
    results[:3], output_dir="./data/",
    parallel=4, verify_checksum=True,
)

# ── RGB bands only — ~150 MB vs 600 MB full scene ─────────────────────────
downloads = client.download(
    results[:5], output_dir="./data/",
    bands=["B02", "B03", "B04"],
)

# ── Full post-processing chain ─────────────────────────────────────────────
downloads = client.download(
    results[:3], output_dir="./processed/",
    parallel=4, retry_attempts=5,
    verify_checksum=True, resume=True,
    bandwidth_limit_mb=20.0,
    post_process=["unzip", "reproject:EPSG:4326",
                  "compress:lzw", "cog"],
)

# ── NDVI workflow: Red + NIR → NDVI → COG ─────────────────────────────────
downloads = client.download(
    results[:3], output_dir="./ndvi/",
    bands=["B04", "B08"],
    post_process=["reproject:EPSG:4326", "ndvi", "cog"],
)

    bbox=(-0.15, 51.47, -0.10, 51.52),
    date_range=("2024-06-01", "2024-06-30"),
    providers=["planetary_computer", "copernicus"],
    cloud_cover_max=10,
)

# 2. Download
downloads = client.download(
    results[:2], output_dir="./data/",
    bands=["B02","B03","B04","B08"],
    post_process=["reproject:EPSG:4326","cog"],
)

# 3. Segment buildings

    output_vector="buildings.geojson")

# 4. Cloud-mask and check quality

# 5. NDVI analysis

# 6. Super-resolve
ga.sr.enhance(downloads[0].path, output_path="sr_4x.tif", scale_factor=4)

# 7. Detect vehicles with natural language

    output_path="vehicles.geojson")

# 8. Caption the scene with VLM

# 9. View results

Error Handling & Resilience

PyGeoVision handles failures at every layer — provider outages, network interruptions, checksum mismatches, GPU OOM, and model load failures are all managed gracefully.

ProviderAuthError
→ pygeovision data auth test PROVIDER
Credentials missing, expired, or rejected. Re-add with data auth add or set the env var.
ProviderTimeoutError
→ Add --timeout 120 or use skip policy
Provider API did not respond in time. Increase timeout or check provider status page.
NoResultsError
→ Widen --cloud-max 30 or extend date range
Search returned zero scenes. Relax filters, check bbox (lon first), or try additional providers.
ChecksumMismatchError
→ --retry 5 --verify-checksum
Downloaded file hash mismatch. Automatic re-download up to retry limit.
GeoAINotAvailableError
→ pip install "pygeovision[geoai]"
GeoAI package not installed. Install the [geoai] extra to use client.geoai.*.
ModelNotFoundError
→ pygeovision models list
Requested model name not in registry. Run models list for valid names or check HF Hub ID.
GPUOutOfMemoryError
→ Reduce --tile-size 256 or --batch-size 4
CUDA OOM during inference or training. Reduce tile size, batch size, or use CPU.
PostProcessError
→ pip install "pygeovision[geo]"
Post-processing chain failed. Usually missing rasterio or GDAL.
KeyringUnavailableError
→ PYTHON_KEYRING_BACKEND=keyring.backends.null.Keyring
No system keyring daemon (Docker, headless SSH). Use environment variables instead.
PipelineStepError
→ --step download to re-run single step
One step of an E2E pipeline failed. Re-run only the failing step after fixing the issue.
Security

Security Model

🔑 Credential Handling
  • Credentials never written to disk in plain text
  • System keyring: macOS Keychain, Windows Credential Manager, Linux Secret Service
  • Encrypted file fallback using Fernet AES-128-CBC
  • All logs redact passwords, tokens, and API keys
  • Credentials cleared from memory after use
🌐 Network Security
  • TLS 1.2+ enforced on all connections
  • SSL certificate verification always on
  • Connection timeouts per provider
  • HTTP proxy support via env vars
  • No telemetry — PyGeoVision does not phone home
📦 Data Integrity
  • SHA256 checksum verification on downloads
  • Atomic file writes — partial downloads never corrupt data
  • Temp files cleaned on failure
  • Download resume prevents data duplication
🔒 Privacy
  • No analytics or usage tracking
  • No third-party requests beyond configured providers
  • All data stays local unless you configure export
  • Model weights downloaded directly from official sources
